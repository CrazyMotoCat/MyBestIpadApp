# MyBestIpadApp

[GitHub repository](https://github.com/CrazyMotoCat/MyBestIpadApp)

## Быстрый старт

```bash
npm ci
npm run dev
```

Проверки проекта:

```bash
npm run check
npm run build
```

GitHub-инфраструктура в репозитории:

- workflow `CI` для `npm run check` и `npm run build`
- issue templates для багов и предложений
- pull request template

Основной стек:

- `React`
- `TypeScript`
- `Vite`
- `PWA`
- `IndexedDB`
- `Service Worker`

Русскоязычный офлайн-блокнот для iPad в формате `PWA` с визуальным направлением “космос + мото”.

Базовая архитектура сохранена:

- `React`
- `TypeScript`
- `Vite`
- `PWA`
- `IndexedDB`
- `Service Worker`

## Кратко

Проект уже представляет собой рабочую локальную продуктовую основу, а не стартовый шаблон.

Сейчас в приложении уже есть:

- главный экран `Мои блокноты`;
- создание и редактирование блокнота;
- фон приложения из пресетов и собственного изображения;
- визуальные стили блокнотов;
- обложки и переплёты;
- каталог бумаги;
- каталог инструментов;
- экран блокнота со страницами и файлами;
- прямое открытие блокнота сразу в рабочую страницу;
- редактор страницы с текстом, рисованием, изображениями, файлами и вставками;
- левая панель редактора с иконками без подписей и сворачиваемыми секциями;
- отдельный режим ластика снизу по центру листа;
- изображения и файлы как элементы страницы с drag/resize;
- корзина рядом с ластиком для удаления объектов перетаскиванием;
- закладки страниц;
- упрощённое перелистывание страниц;
- локальное сохранение всего этого в `IndexedDB`.

## Краткое ревью

Сильные стороны текущего состояния:

- архитектура осталась модульной и не была переписана с нуля;
- данные, визуальные пресеты и настройки вынесены из UI в конфиги и типы;
- `IndexedDB` уже хранит не только базовые записи, но и реальные продуктовые сущности;
- интерфейс на русском языке и уже соответствует продуктовой идее;
- `npm run check` и `npm run build` проходят.

Что особенно хорошо заложено:

- расширяемая модель бумаги;
- расширяемая модель инструментов;
- расширяемая модель фонов приложения;
- расширяемая модель обложек и переплётов;
- локальное хранение `Blob` для изображений, файлов, фонов и обложек;
- левая функциональная колонка внутри редактора;
- разделение по `features`, `shared/config`, `shared/types`, `shared/lib`, `shared/ui`.

Что ещё не доведено до финального продукта:

- нет полноценного свободного layout-редактора для всех типов элементов;
- shape notes уже можно вставлять, двигать и растягивать, но изображения и файлы ещё не имеют такого же уровня управления;
- анимация перелистывания намеренно лёгкая, а не тяжёлая 3D page curl;
- `Service Worker` пока базовый и потребует отдельной полировки для более сложного офлайн-кеша;
- ограничения браузерной квоты, особенно в iPad Safari, остаются важным фактором для крупных вложений.

## Что реализовано

### 1. Главный экран

- крупный экран `Мои блокноты`;
- кнопки:
  - `Создать`
  - `Настроить фон приложения`
- визуальные карточки блокнотов;
- отображение выбранной обложки и переплёта прямо на карточке.

### 2. Фон приложения

Добавлены:

- карточки-превью пресетов фона;
- выбор не по цвету, а по визуальному образу;
- загрузка собственного изображения фона;
- локальное сохранение пользовательского фона;
- восстановление после перезагрузки;
- затемняющий overlay для сохранения читаемости интерфейса.

### 3. Создание блокнота

В модалке создания блокнота теперь можно выбрать:

- название;
- цвет акцента;
- стиль блокнота;
- тип блокнота;
- тип бумаги;
- цвет бумаги;
- инструмент по умолчанию;
- обложку;
- переплёт;
- свою картинку для обложки.

### 4. Стили блокнота

Стили теперь отображаются как визуальные карточки, а не как абстрактный цвет.

На карточках есть:

- символ;
- декоративная поверхность;
- образ/характер пресета;
- визуально понятный предпросмотр.

### 5. Типы блокнотов

Список типов расширен. Помимо уже существующих добавлены:

- ежедневник;
- блокнот с пружиной;
- планер;
- записная книжка.

### 6. Обложки и внешний вид блокнота

Добавлены:

- пресеты обложек;
- выбор готовой обложки;
- загрузка своей картинки для обложки;
- отображение обложки на главном экране;
- отображение той же обложки при открытии блокнота;
- редактирование оформления уже созданного блокнота.

### 7. Переплёт

Добавлен выбор переплёта:

- пружина;
- скрепка;
- кольца.

Переплёт:

- выбирается при создании;
- отображается на карточке блокнота;
- отображается на открытом блокноте;
- влияет на визуальный макет, а не только на текстовые данные.

### 8. Бумага

Каталог бумаги остаётся расширяемым и вынесенным в конфиги.

Поддержаны:

- чистая;
- линованная;
- в клетку;
- в точку;
- миллиметровка;
- инженерная сетка;
- изометрическая;
- музыкальная;
- каллиграфическая;
- для скетчинга;
- крафтовая;
- кремовая;
- тёмная;
- ежедневник;
- планер;
- чек-лист;
- Cornell notes;
- storyboard;
- комикс-разметка;
- школьная;
- лабораторная;
- архитектурная;
- чертёжная.

Также добавлен отдельный выбор цвета листа без потери текстуры и разметки бумаги.

### 9. Инструменты

Поддержан широкий каталог пресетов:

- шариковая ручка;
- гелевая ручка;
- перьевая ручка;
- капиллярная ручка;
- линер;
- маркер;
- текстовыделитель;
- карандаш;
- механический карандаш;
- кисть;
- каллиграфическая кисть;
- тонкая кисть;
- толстая кисть;
- фломастер;
- уголь;
- ластик.

Важно:

- `default tool` существует только как стартовый инструмент;
- после открытия страницы можно свободно переключаться между всеми пресетами;
- нет искусственного ограничения “один инструмент на весь блокнот”.

### 10. Левая боковая колонка в редакторе

Добавлена вертикальная колонка слева с разделами:

- вставки;
- ручки;
- кисти и карандаши;
- лист;
- закладки.

### 11. Вставки / фигуры

Добавлена основа для `shapeNote` элементов:

- прямоугольник;
- квадрат;
- круг/овал;
- облачко/выноска;
- стикер;
- бумажная заметка;
- карточка.

Для вставок поддержано:

- выбор формы;
- вставка на страницу;
- изменение размера;
- перемещение;
- выбор цвета;
- выбор стиля бумаги внутри;
- выбор края:
  - ровный;
  - оторванный.

### 12. Закладки

Добавлены:

- `isBookmarked` на уровне страницы;
- кнопка-звезда в правом нижнем углу страницы;
- локальное сохранение закладок;
- список закладок через левую боковую колонку;
- быстрый переход к отмеченным страницам.

### 13. Перелистывание страниц

Реализован облегчённый вариант page flipping:

- на touch-устройствах листание работает через свайп;
- на компьютере работает через клики по нижним углам страницы;
- анимация лёгкая и не перегружает интерфейс.

### 14. Изображения и файлы

Поддержано:

- добавление изображений на страницу;
- добавление файлов на страницу;
- добавление файлов в блокнот;
- локальное хранение в `IndexedDB`;
- восстановление после перезагрузки;
- офлайн-доступ.

## Ограничения и заметки

- локальное хранение пользовательских изображений и файлов ограничено браузерной квотой;
- на iPad Safari это особенно важно для крупных вложений;
- фоны и обложки с пользовательскими картинками сохраняются локально через `Blob`;
- перелистывание реализовано как облегчённая 2D-анимация, а не тяжёлый 3D curl;
- shape inserts уже двигаются и растягиваются, но полноценный свободный layout для всех видов элементов ещё не завершён;
- `Service Worker` пока базовый и не покрывает продвинутую стратегию кеширования всех пользовательских asset-сценариев.

## Следующие логичные шаги

- добавить свободное перемещение и resize для изображений и файлов;
- довести редактор до полноценного layout-режима для всех элементов страницы;
- добавить undo/redo для рисования и манипуляций с элементами;
- улучшить рисование: больше слоёв, richer brush behavior, более глубокая настройка;
- доработать перелистывание и навигацию по страницам;
- усилить офлайн-кеш и поведение на iPad Safari;
- при необходимости заменить временные графические ассеты на отдельные растровые/иллюстративные ресурсы.

## Текущий прогресс по ROADMAP

### Шаг 1. Основа для настроек фона

Выполнено в текущем цикле правок:

- модель `AppSettings` расширена полями `backgroundDimAmount` и `backgroundBlurAmount`;
- сохранена совместимость со старым локальным полем `overlayOpacity` через нормализацию legacy-данных;
- `AppShell` теперь применяет отдельно затемнение и размытие фоновой сцены;
- в модалке настройки фона добавлены два независимых ползунка:
  - сила затемнения;
  - сила размытия;
- новые значения продолжают храниться локально в `IndexedDB` и восстанавливаются после перезагрузки.

Следующий запланированный шаг:

- переделка главного экрана и карточек блокнотов в более реалистичный визуальный макет.

### Шаг 2. Реалистичные блокноты на главном экране

Выполнено в текущем цикле правок:

- главный экран переведён в режим галереи блокнотов, а не набора обычных веб-карточек;
- карточка блокнота переработана в визуальный объект с:
  - толщиной;
  - страницами;
  - корешком;
  - обложкой;
  - переплётом;
- с главного экрана убраны служебные метаданные на самих блокнотах:
  - тип;
  - бумага;
  - инструмент;
  - дата обновления;
- название теперь показывается только по центру обложки и только если оно задано;
- добавлена более атмосферная сцена-галерея для блока со списком блокнотов.

Следующий запланированный шаг:

- доведение фоновых сцен приложения: визуальная глубина пресетов, финальная настройка `dim/blur` и полировка сцен-предпросмотра.

### Шаг 3. Фоновые сцены и превью

Выполнено в текущем цикле правок:

- пресеты фона приложения переработаны в более атмосферные сцены вместо простых абстрактных градиентов;
- отдельно усилены сцены:
  - `Глубокий космос`;

- выбор фона теперь лучше соответствует требованию roadmap про “реальные сцены-превью”, а не просто цветовые пятна.

Следующий запланированный шаг:

- переход к более визуальному и цельному экрану создания блокнота, без смены архитектуры и хранения.

### Шаг 4. Визуальное создание блокнота

Выполнено в текущем цикле правок:

- модалка создания блокнота перестроена в более цельный визуальный flow;
- добавлен живой предпросмотр будущего блокнота прямо внутри формы;

  - внешний вид;

Следующий запланированный шаг:

- переход к сценарию прямого открытия блокнота сразу в рабочую страницу, без промежуточного первого экрана.

### Шаг 5. Прямое открытие блокнота в рабочую страницу

Выполнено в текущем цикле правок:

- основной маршрут `notebooks/:notebookId` больше не показывает промежуточный экран со списком страниц;
- при открытии блокнота приложение теперь сразу ищет первую страницу и переводит пользователя в редактор;
- если в блокноте ещё нет страниц, первая рабочая страница создаётся автоматически и наследует формат блокнота;
- обзор блокнота со страницами, вложениями и настройкой оформления сохранён как отдельный вспомогательный маршрут управления;
- после создания нового блокнота пользователь тоже сразу попадает в рабочую страницу, а не возвращается на промежуточный список.

Следующий запланированный шаг:

- очистка каркаса редактора: левая панель без подписей, более чистый экран страницы и подготовка к режиму ластика.

### Шаг 6. Каркас редактора и левая панель

Выполнено в текущем цикле правок:

- левая панель редактора переведена в режим иконок без текстовых подписей;
- повторное нажатие по иконке теперь сворачивает активную секцию, а не держит панель постоянно раскрытой;
- активная секция открывается как отдельный рельефный боковой блок с компактным заголовком и кнопкой закрытия;
- верх редактора стал чище: вместо служебной строки используется более компактный набор статусов и параметров страницы;
- сама страница стала менее “сайтовой”: убраны лишние подписи над текстом и рисованием, а вложения перенесены в более компактный нижний ряд поддержки.

Следующий запланированный шаг:

- усиление самого редактора: режим ластика, дальнейшая чистка листа и подготовка к свободному layout-редактированию элементов.

### Шаг 7. Вертикальная rail-панель и режим ластика

Выполнено в текущем цикле правок:

- левая панель иконок сохранена именно слева и именно вертикально, без ухода в горизонтальную строку сверху на узких размерах;
- для сенсорного сценария панель остаётся отдельной левой колонкой, а не превращается в верхнюю полосу;
- в редактор добавлен отдельный переключатель `Ластик` снизу по центру листа;
- ластик работает как самостоятельный режим рисования и переключается обратно к предыдущему инструменту повторным нажатием.

Следующий запланированный шаг:

- переход к свободному layout-редактированию более сложных элементов страницы, прежде всего изображений и файлов.

### Шаг 8. Layout для изображений и файлов + спокойное autosave

Выполнено в текущем цикле правок:

- изображения и файлы теперь выводятся прямо на лист как элементы страницы, а не только как нижние списки;
- для изображений и файлов добавлены перемещение и изменение размера внутри страницы;
- у изображений появилась подпись, у файлов заметка, и оба поля продолжают жить в локальной модели данных;
- статус сохранения больше не уходит в бесконечное мигание: autosave перестал сам себя перезапускать из-за обновления `page.updatedAt`;
- надпись `Сохраняем...` теперь появляется только в момент реальной записи, а не при каждом локальном изменении до срабатывания debounce.

Следующий запланированный шаг:

- дальнейшее развитие свободного layout-редактирования и полировка наложения элементов на листе.

### Шаг 9. Плавный drag и корзина на листе

Выполнено в текущем цикле правок:

- перемещение и resize вставок, изображений и файлов больше не тормозят экран из-за записи в IndexedDB на каждом движении;
- drag теперь идёт через локальный черновой state внутри слоёв, а запись в базу происходит только по завершению перемещения или resize;
- текст shape-вставок, подписи изображений и заметки файлов коммитятся по `blur`, без лишних тяжёлых записей во время самого drag;
- рядом с ластиком добавлена корзина;
- объект на листе можно перетащить в корзину и удалить;
- корзина подсвечивается, когда в неё заносят объект.

Следующий запланированный шаг:

- дальнейшая полировка свободного layout-режима и работа со слоями/наложением элементов страницы.
### Шаг 10. Долгое касание и лист без верхнего блока

Выполнено в текущем цикле правок:

- верхний блок редактора с названием страницы и служебными действиями полностью убран;
- название страницы, макет и статус сохранения перенесены в левую секцию `Лист` и в компактную плавающую плашку на самом листе;
- изображения, файлы и shape-вставки теперь можно брать долгим касанием прямо на объекте, без отдельного мгновенного drag-сценария для iPad;
- после долгого касания объект продолжает нормально переноситься в корзину рядом с ластиком;
- для touch-сценария поведение объектов стало ближе к реальному рабочему жесту, а верх страницы перестал занимать место и отвлекать от самого листа.

Следующий запланированный шаг:

- дальнейшая полировка свободного layout-режима: слои, точность наложения и более аккуратная визуальная иерархия элементов страницы.
### Шаг 11. Единый лист и полировка слоёв

Выполнено в текущем цикле правок:

- редактор переведён на единый лист бумаги: текст, рисунок Pencil и свободные объекты теперь живут в одной рабочей области;

- Apple Pencil теперь рисует прямо по листу поверх текста, а клавиатурный текст продолжает вводиться в тот же лист без отдельного нижнего canvas-блока;
- активный объект поднимается выше остальных, поэтому наложение изображений, файлов и вставок стало понятнее и аккуратнее;
- для активных объектов добавлена более явная визуальная иерархия через выделение и усиленную глубину слоя.

Следующий запланированный шаг:

- дальнейшая полировка notebook-flow и перелистывания;
- более умное поведение страниц при переходе к следующему листу и автосоздании новых страниц.
### Шаг 12. Фикс листа и закрытая rail-панель при входе

Выполнено в текущем цикле правок:

- восстановлена нормальная высота рабочего листа после перехода на единый paper-stage, так что при открытии страницы теперь виден сам лист, а не только окружающий интерфейс;
- у листа снова есть стабильная рабочая область по высоте и на desktop, и на iPad-ширинах;
- левая вертикальная колонка больше не раскрывается автоматически при входе в блокнот или страницу;
- при нажатии на блокнот пользователь по-прежнему сразу попадает на рабочий лист, но секции rail-панели остаются закрытыми, пока не нажать на конкретную иконку;
- сценарий открытия стал чище: сначала лист, затем уже дополнительные инструменты по явному действию пользователя.

Следующий запланированный шаг:

- дальнейшая полировка перелистывания;
- более естественный переход к следующему листу и автосоздание страниц без лишних промежуточных состояний.
Выполненный шаг 13:

Следующий шаг по выбранной очередности:

Выполненный шаг 14:
- исправлена высотная цепочка редактора от `AppShell` до `editor-sheet`, поэтому лист теперь растягивается на всю доступную высоту окна и доходит до нижней границы экрана;
- для редактора убран лишний верхний отступ секции, из-за которого лист визуально "висел" выше и терял нижнюю часть рабочей области;

- нижняя часть листа дополнительно получила мягкую подложку и тень, чтобы страница ощущалась как физический лист, лежащий в блокноте.

Следующий шаг по выбранной очередности:
- дальнейшая полировка page-turn UX на iPad;
- более аккуратная работа с длинным текстом на границе листа и подготовка к ещё более натуральному сценарию последовательных страниц.
Багфикс по высоте листа:

- рабочий лист переведён в режим геометрии A4: редактор теперь вычисляет размер листа по реальной доступной области, а не полагается только на цепочку `height: 100%`;

Полноэкранный фикс листа:
