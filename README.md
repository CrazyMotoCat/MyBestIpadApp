# MyBestIpadApp

Русскоязычное офлайн-приложение-блокнот для iPad в формате PWA.

Проект задуман как красивый личный notebook experience с акцентом на:
- работу без интернета;
- локальное хранение данных на устройстве;
- touch-friendly интерфейс для iPad;
- атмосферный визуальный стиль в теме «космос + мото»;
- редактирование страниц с текстом, рисованием и вложениями.

## Текущее состояние

Проект уже вышел за пределы стартового шаблона и представляет собой рабочую локальную продуктовую основу.

Сейчас в приложении есть:
- экран **«Мои блокноты»**;
- создание и редактирование блокнотов;
- выбор фона приложения из пресетов и собственного изображения;
- визуальные стили блокнотов;
- обложки и переплёты;
- каталог бумаги;
- каталог инструментов;
- экран блокнота со страницами и файлами;
- прямое открытие блокнота в рабочую страницу;
- редактор страницы с текстом, рисованием, изображениями, файлами и вставками;
- drag/resize для части объектов на странице;
- корзина для удаления объектов перетаскиванием;
- закладки страниц;
- локальное сохранение через IndexedDB.

## Что важно для продукта

### iPad-first, но без потери desktop usability
Интерфейс в первую очередь ориентирован на iPad и touch-сценарии, но при этом должен оставаться удобным и на обычном компьютере.

### Offline-first
Приложение должно оставаться полезным без сервера и без обязательного интернет-соединения. Основное состояние хранится локально.

### Не просто сайт, а notebook experience
Ключевая цель проекта — не обычный CRUD-интерфейс, а ощущение цельного красивого блокнота с атмосферой, визуальным характером и понятным рабочим редактором.

## Архитектурные ориентиры

Проект уже использует модульное разбиение, которое стоит сохранить:
- `app` — каркас и маршруты;
- `features` — пользовательские сценарии;
- `shared/config` — каталоги пресетов и визуальных сущностей;
- `shared/types` — доменные типы;
- `shared/lib` — инфраструктура и локальное хранение;
- `shared/ui` — общие UI-компоненты.

При дальнейшем развитии желательно:
- не собирать всё в одном гигантском editor-компоненте;
- выносить новые сущности в типы и конфиги, а не зашивать в UI;
- держать отдельными canvas-логику, page-layout, media-вложения и persistence.

## Ограничения текущей версии

На текущем baseline ещё важно учитывать:
- сервис-воркер и офлайн-кеш требуют дальнейшей полировки;
- ограничения браузерной квоты, особенно в iPad Safari, важны для тяжёлых вложений;
- свободный layout-редактор для всех типов элементов ещё не доведён до одинакового уровня зрелости;
- drag/resize и управление разными типами page-элементов пока не полностью выровнены;
- проекту ещё нужен следующий проход по архитектуре редактора и по quality-слою around offline/persistence.

## Что не входит в ближайший scope

В ближайшую базовую линию проекта не закладываются:
- аккаунты и регистрация;
- облачная синхронизация;
- совместное редактирование;
- сложный редактор уровня Notion;
- публикация в App Store как нативного приложения.
