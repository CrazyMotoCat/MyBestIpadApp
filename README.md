# MyBestIpadApp

Русскоязычный офлайн-блокнот для iPad в формате `PWA` с визуальным направлением “космос + мото”.

Базовая архитектура сохранена:

- `React`
- `TypeScript`
- `Vite`
- `PWA`
- `IndexedDB`
- `Service Worker`

## Кратко

Проект уже представляет собой рабочую локальную продуктовую основу, а не стартовый шаблон.

Сейчас в приложении уже есть:

- главный экран `Мои блокноты`;
- создание и редактирование блокнота;
- фон приложения из пресетов и собственного изображения;
- визуальные стили блокнотов;
- обложки и переплёты;
- каталог бумаги;
- каталог инструментов;
- экран блокнота со страницами и файлами;
- редактор страницы с текстом, рисованием, изображениями, файлами и вставками;
- закладки страниц;
- упрощённое перелистывание страниц;
- локальное сохранение всего этого в `IndexedDB`.

## Краткое ревью

Сильные стороны текущего состояния:

- архитектура осталась модульной и не была переписана с нуля;
- данные, визуальные пресеты и настройки вынесены из UI в конфиги и типы;
- `IndexedDB` уже хранит не только базовые записи, но и реальные продуктовые сущности;
- интерфейс на русском языке и уже соответствует продуктовой идее;
- `npm run check` и `npm run build` проходят.

Что особенно хорошо заложено:

- расширяемая модель бумаги;
- расширяемая модель инструментов;
- расширяемая модель фонов приложения;
- расширяемая модель обложек и переплётов;
- локальное хранение `Blob` для изображений, файлов, фонов и обложек;
- левая функциональная колонка внутри редактора;
- разделение по `features`, `shared/config`, `shared/types`, `shared/lib`, `shared/ui`.

Что ещё не доведено до финального продукта:

- нет полноценного свободного layout-редактора для всех типов элементов;
- shape notes уже можно вставлять, двигать и растягивать, но изображения и файлы ещё не имеют такого же уровня управления;
- анимация перелистывания намеренно лёгкая, а не тяжёлая 3D page curl;
- `Service Worker` пока базовый и потребует отдельной полировки для более сложного офлайн-кеша;
- ограничения браузерной квоты, особенно в iPad Safari, остаются важным фактором для крупных вложений.

## Что реализовано

### 1. Главный экран

- крупный экран `Мои блокноты`;
- кнопки:
  - `Создать`
  - `Настроить фон приложения`
- визуальные карточки блокнотов;
- отображение выбранной обложки и переплёта прямо на карточке.

### 2. Фон приложения

Добавлены:

- карточки-превью пресетов фона;
- выбор не по цвету, а по визуальному образу;
- загрузка собственного изображения фона;
- локальное сохранение пользовательского фона;
- восстановление после перезагрузки;
- затемняющий overlay для сохранения читаемости интерфейса.

### 3. Создание блокнота

В модалке создания блокнота теперь можно выбрать:

- название;
- цвет акцента;
- стиль блокнота;
- тип блокнота;
- тип бумаги;
- цвет бумаги;
- инструмент по умолчанию;
- обложку;
- переплёт;
- свою картинку для обложки.

### 4. Стили блокнота

Стили теперь отображаются как визуальные карточки, а не как абстрактный цвет.

На карточках есть:

- символ;
- декоративная поверхность;
- образ/характер пресета;
- визуально понятный предпросмотр.

### 5. Типы блокнотов

Список типов расширен. Помимо уже существующих добавлены:

- ежедневник;
- блокнот с пружиной;
- планер;
- записная книжка.

### 6. Обложки и внешний вид блокнота

Добавлены:

- пресеты обложек;
- выбор готовой обложки;
- загрузка своей картинки для обложки;
- отображение обложки на главном экране;
- отображение той же обложки при открытии блокнота;
- редактирование оформления уже созданного блокнота.

### 7. Переплёт

Добавлен выбор переплёта:

- пружина;
- скрепка;
- кольца.

Переплёт:

- выбирается при создании;
- отображается на карточке блокнота;
- отображается на открытом блокноте;
- влияет на визуальный макет, а не только на текстовые данные.

### 8. Бумага

Каталог бумаги остаётся расширяемым и вынесенным в конфиги.

Поддержаны:

- чистая;
- линованная;
- в клетку;
- в точку;
- миллиметровка;
- инженерная сетка;
- изометрическая;
- музыкальная;
- каллиграфическая;
- для скетчинга;
- крафтовая;
- кремовая;
- тёмная;
- ежедневник;
- планер;
- чек-лист;
- Cornell notes;
- storyboard;
- комикс-разметка;
- школьная;
- лабораторная;
- архитектурная;
- чертёжная.

Также добавлен отдельный выбор цвета листа без потери текстуры и разметки бумаги.

### 9. Инструменты

Поддержан широкий каталог пресетов:

- шариковая ручка;
- гелевая ручка;
- перьевая ручка;
- капиллярная ручка;
- линер;
- маркер;
- текстовыделитель;
- карандаш;
- механический карандаш;
- кисть;
- каллиграфическая кисть;
- тонкая кисть;
- толстая кисть;
- фломастер;
- уголь;
- ластик.

Важно:

- `default tool` существует только как стартовый инструмент;
- после открытия страницы можно свободно переключаться между всеми пресетами;
- нет искусственного ограничения “один инструмент на весь блокнот”.

### 10. Левая боковая колонка в редакторе

Добавлена вертикальная колонка слева с разделами:

- вставки;
- ручки;
- кисти и карандаши;
- лист;
- закладки.

### 11. Вставки / фигуры

Добавлена основа для `shapeNote` элементов:

- прямоугольник;
- квадрат;
- круг/овал;
- облачко/выноска;
- стикер;
- бумажная заметка;
- карточка.

Для вставок поддержано:

- выбор формы;
- вставка на страницу;
- изменение размера;
- перемещение;
- выбор цвета;
- выбор стиля бумаги внутри;
- выбор края:
  - ровный;
  - оторванный.

### 12. Закладки

Добавлены:

- `isBookmarked` на уровне страницы;
- кнопка-звезда в правом нижнем углу страницы;
- локальное сохранение закладок;
- список закладок через левую боковую колонку;
- быстрый переход к отмеченным страницам.

### 13. Перелистывание страниц

Реализован облегчённый вариант page flipping:

- на touch-устройствах листание работает через свайп;
- на компьютере работает через клики по нижним углам страницы;
- анимация лёгкая и не перегружает интерфейс.

### 14. Изображения и файлы

Поддержано:

- добавление изображений на страницу;
- добавление файлов на страницу;
- добавление файлов в блокнот;
- локальное хранение в `IndexedDB`;
- восстановление после перезагрузки;
- офлайн-доступ.

## Ограничения и заметки

- локальное хранение пользовательских изображений и файлов ограничено браузерной квотой;
- на iPad Safari это особенно важно для крупных вложений;
- фоны и обложки с пользовательскими картинками сохраняются локально через `Blob`;
- перелистывание реализовано как облегчённая 2D-анимация, а не тяжёлый 3D curl;
- shape inserts уже двигаются и растягиваются, но полноценный свободный layout для всех видов элементов ещё не завершён;
- `Service Worker` пока базовый и не покрывает продвинутую стратегию кеширования всех пользовательских asset-сценариев.

## Следующие логичные шаги

- добавить свободное перемещение и resize для изображений и файлов;
- довести редактор до полноценного layout-режима для всех элементов страницы;
- добавить undo/redo для рисования и манипуляций с элементами;
- улучшить рисование: больше слоёв, richer brush behavior, более глубокая настройка;
- доработать перелистывание и навигацию по страницам;
- усилить офлайн-кеш и поведение на iPad Safari;
- при необходимости заменить временные графические ассеты на отдельные растровые/иллюстративные ресурсы.

## Текущий прогресс по ROADMAP

### Шаг 1. Основа для настроек фона

Выполнено в текущем цикле правок:

- модель `AppSettings` расширена полями `backgroundDimAmount` и `backgroundBlurAmount`;
- сохранена совместимость со старым локальным полем `overlayOpacity` через нормализацию legacy-данных;
- `AppShell` теперь применяет отдельно затемнение и размытие фоновой сцены;
- в модалке настройки фона добавлены два независимых ползунка:
  - сила затемнения;
  - сила размытия;
- новые значения продолжают храниться локально в `IndexedDB` и восстанавливаются после перезагрузки.

Следующий запланированный шаг:

- переделка главного экрана и карточек блокнотов в более реалистичный визуальный макет.

### Шаг 2. Реалистичные блокноты на главном экране

Выполнено в текущем цикле правок:

- главный экран переведён в режим галереи блокнотов, а не набора обычных веб-карточек;
- карточка блокнота переработана в визуальный объект с:
  - толщиной;
  - страницами;
  - корешком;
  - обложкой;
  - переплётом;
- с главного экрана убраны служебные метаданные на самих блокнотах:
  - тип;
  - бумага;
  - инструмент;
  - дата обновления;
- название теперь показывается только по центру обложки и только если оно задано;
- добавлена более атмосферная сцена-галерея для блока со списком блокнотов.

Следующий запланированный шаг:

- доведение фоновых сцен приложения: визуальная глубина пресетов, финальная настройка `dim/blur` и полировка сцен-предпросмотра.

### Шаг 3. Фоновые сцены и превью

Выполнено в текущем цикле правок:

- пресеты фона приложения переработаны в более атмосферные сцены вместо простых абстрактных градиентов;
- отдельно усилены сцены:
  - `Глубокий космос`;
  - `Метеоритный поток`;
  - `Ночная трасса`;
  - `Северный двигатель`;
  - `Пыль Сатурна`;
  - `Карбоновый дрифт`;
- карточки выбора фона стали крупнее и более “сценическими”, с акцентом на превью;
- блок настроек `dim/blur` визуально отделён и стал понятнее в модалке;
- выбор фона теперь лучше соответствует требованию roadmap про “реальные сцены-превью”, а не просто цветовые пятна.

Следующий запланированный шаг:

- переход к более визуальному и цельному экрану создания блокнота, без смены архитектуры и хранения.

### Шаг 4. Визуальное создание блокнота

Выполнено в текущем цикле правок:

- модалка создания блокнота перестроена в более цельный визуальный flow;
- добавлен живой предпросмотр будущего блокнота прямо внутри формы;
- форма разделена на смысловые секции:
  - основа;
  - стиль и формат;
  - внешний вид;
  - рабочая среда;
- выбор обложки, переплёта, бумаги и стартового инструмента остался на уже существующих сущностях и API, без ломки архитектуры;
- пользователь теперь видит образ блокнота до сохранения, а не просто набор разрозненных полей.

Следующий запланированный шаг:

- переход к сценарию прямого открытия блокнота сразу в рабочую страницу, без промежуточного первого экрана.
