# Sprint 1 Smoke Pack

Практический чеклист для финальной проверки Sprint 1 вокруг `v1.2 — Offline Reliability`.

## Перед началом
- Открыть приложение на iPad в Safari.
- Убедиться, что используется свежая сборка и сервис-воркер уже обновился.
- Для обновления использовать только пользовательский путь: `update available -> apply update -> reload`.

## Чеклист

### 1. Первый онлайн-запуск
- [ ] Открыть приложение в Safari при активной сети.
- [ ] Дождаться загрузки shell и появления offline coach или `PWA статус`.
- [ ] Убедиться, что приложение не зависает на пустом экране.

### 2. Прогрев shell завершён
- [ ] Дождаться подтверждения, что shell готов.
- [ ] Проверить, что `PWA статус` показывает один понятный итог по offline readiness.
- [ ] Убедиться, что баннер перезагрузки не появляется без реального обновления.

### 3. Закрытие и повторное открытие
- [ ] Закрыть вкладку или выйти из приложения.
- [ ] Открыть приложение снова.
- [ ] Проверить, что shell возвращается без ручной очистки кэша.

### 4. Перезапуск в авиарежиме
- [ ] Включить на iPad авиарежим.
- [ ] Запустить приложение заново после уже выполненного прогрева shell.
- [ ] Проверить, что приложение открывается без сети и не падает на старте.

### 5. Запуск с Home Screen
- [ ] Запустить приложение с иконки на домашнем экране iPad.
- [ ] Проверить, что поведение соответствует ожидаемому standalone path.
- [ ] Убедиться, что статус не противоречит запуску из Safari.

### 6. Появление обновления
- [ ] Подготовить новую версию SW или свежую сборку.
- [ ] Дождаться сигнала `update available`.
- [ ] Убедиться, что у пользователя есть явный CTA на применение обновления.

### 7. Перезагрузка после обновления
- [ ] Нажать `Применить обновление`.
- [ ] Дождаться состояния, где новая оболочка уже готова.
- [ ] Нажать `Перезагрузить`.
- [ ] Проверить, что старый экран не продолжает жить поверх нового shell-контракта.

### 8. Восстановление прерванной сессии
- [ ] Начать редактировать страницу.
- [ ] Оставить незавершённые изменения.
- [ ] Перезагрузить приложение или закрыть и открыть его снова.
- [ ] Проверить, что draft/recovery поведение предсказуемо и понятно пользователю.

## Критерии прохождения
- Офлайн-запуск работает без доступа к сети.
- Update flow всегда требует видимого действия пользователя.
- После обновления нет полу-старого UI поверх нового shell.
- Прерванная сессия либо восстанавливается явно, либо понятно объясняет, что именно восстановлено.

## Критерии остановки
- Любой шаг, который приводит к пустому экрану, ложному `ready`-сигналу или тихой потере данных, немедленно останавливает smoke-check и заводится как баг.
