# MyBestIpadApp

## Кратко

Это личное PWA-приложение для iPad в формате красивого офлайн-блокнота.
Пользователь должен иметь возможность:

- создавать свои блокноты;
- создавать и редактировать страницы;
- писать текст;
- рисовать пальцем или стилусом;
- настраивать внешний вид страниц;
- пользоваться приложением без интернета.

## Почему выбран PWA

Изначально рассматривалась нативная iPad-разработка, но она требует macOS и Xcode.
Так как работа ведется с Windows, выбран путь через PWA:

- приложение можно разрабатывать на Windows;
- его можно открыть на iPad через Safari;
- его можно добавить на домашний экран как приложение;
- можно реализовать офлайн-режим;
- можно хранить данные локально на устройстве.

## Основной стек

- Язык: `TypeScript`
- UI: `React`
- Сборка: `Vite`
- Формат приложения: `PWA`
- Локальное хранилище: `IndexedDB`
- Оффлайн-режим: `Service Worker`
- Рисование: `HTML Canvas` или библиотека поверх canvas

## Что не делаем в первой версии

- регистрацию и аккаунты;
- облачную синхронизацию;
- совместное редактирование;
- сложный редактор уровня Notion;
- публикацию в App Store.

## MVP

Первая рабочая версия должна уметь:

1. создавать блокноты;
2. показывать список блокнотов;
3. открывать блокнот;
4. создавать страницы внутри блокнота;
5. редактировать страницу;
6. добавлять текст на страницу;
7. рисовать на странице;
8. менять фон, цвет и базовое оформление страницы;
9. сохранять все локально и работать без интернета.

## Экраны MVP

- экран списка блокнотов;
- экран блокнота со списком страниц;
- экран редактора страницы.

## Предлагаемая структура проекта

- `src/app` - маршруты и каркас приложения
- `src/features/notebooks` - логика блокнотов
- `src/features/pages` - логика страниц
- `src/features/editor` - редактор страницы
- `src/features/drawing` - рисование
- `src/shared/ui` - общие UI-компоненты
- `src/shared/lib/db` - работа с IndexedDB
- `src/shared/lib/pwa` - manifest, service worker и офлайн-логика

## Технический порядок работ

1. Создать новый проект на `Vite + React + TypeScript`.
2. Настроить PWA: `manifest`, иконки, offline cache, installability.
3. Подключить `IndexedDB` для локального хранения.
4. Реализовать список блокнотов.
5. Реализовать список страниц.
6. Реализовать редактор страницы.
7. Добавить рисование.
8. Добавить оформление страниц.
9. Проверить установку на iPad через Safari.

## На что обратить внимание

- iPad Safari имеет ограничения по сравнению с нативным приложением;
- офлайн-режим должен быть предусмотрен с самого начала;
- большие бинарные данные нужно хранить аккуратно;
- автосохранение важнее сложных декоративных функций;
- сначала нужна надежная основа, потом украшения.
