# Sprint 3 Smoke Pack

Этот smoke-pack фиксирует минимальный критический editor flow для `Sprint 3 / editor quality gates`.

## Preconditions

- запускать на свежей сборке из актуального `main`;
- проверить сценарий в desktop Chrome и, если возможно, на iPad Safari path;
- убедиться, что в браузере доступен обычный local-first режим, без DevTools и ручных правок IndexedDB.

## Minimal Happy Path

1. Открыть приложение и дождаться списка блокнотов.
2. Создать новый блокнот с уникальным названием.
3. Перейти в первый лист блокнота.
4. Один раз нажать по рабочему листу, чтобы появился текстовый блок.
5. Ввести короткий текст и сохранить его.
6. Перезагрузить страницу.
7. Убедиться, что текст и сам блокнот восстановились без потери данных.

## Recovery Checks

- если появится offline coach или update banner, его нужно явно закрыть или пройти по CTA, а не игнорировать;
- если после reload текст не сохраняется, это блокер для Sprint 3 done;
- если editor не создаёт текстовый блок по клику на лист, happy path считается сломанным.

## Done Criteria

Sprint 3 можно считать закрытым только если:
- минимальный happy-path проходит автоматически в e2e;
- CI умеет запускать этот e2e job;
- smoke-pack выше остаётся как ручная fallback-проверка для editor flow.
