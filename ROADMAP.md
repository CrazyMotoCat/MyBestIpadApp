Продолжай развитие текущего приложения-блокнота для iPad. Не переписывай проект с нуля и не ломай существующую архитектуру. Сохрани текущую основу: React + TypeScript + Vite, PWA-формат, офлайн-логику, локальное хранение через IndexedDB, текущую feature-структуру и уже существующие базовые сценарии.

Нужен не временный прототип, а аккуратная, расширяемая и цельная реализация, которая визуально и по UX ощущается как настоящий красивый блокнот, а не как сайт.

==================================================
ГЛАВНАЯ ЦЕЛЬ
==================================================

Превратить текущее приложение в красивый, атмосферный, русскоязычный офлайн-блокнот для iPad в смешанной стилистике:
- космос;
- мото;
- современный визуальный стиль;
- ощущение реального блокнота;
- удобное сенсорное управление;
- поддержка текста, рисования, вставок, фото, файлов и перелистывания страниц.

==================================================
ОБЯЗАТЕЛЬНЫЕ ОГРАНИЧЕНИЯ
==================================================

1. Не ломай существующую архитектуру проекта.
2. Сохрани:
- React + TypeScript + Vite;
- PWA;
- офлайн-работу;
- IndexedDB;
- текущую структуру app/features/shared;
- уже существующие возможности, если они не противоречат новому UX.
3. Все тексты интерфейса должны быть на русском языке.
4. Интерфейс должен быть адаптирован под iPad и сенсорное управление.
5. Одновременно он должен оставаться usable на компьютере с мышкой.
6. Не делай все в одном гигантском компоненте.
7. Все новые сущности, конфиги, ассеты и UI-блоки нужно разложить по логичной архитектуре.

==================================================
ОБЩАЯ ВИЗУАЛЬНАЯ КОНЦЕПЦИЯ
==================================================

Сделай единый визуальный стиль приложения в теме “космос + мото”.

Важно:
- темная база интерфейса;
- хорошая читаемость;
- атмосферные фоны;
- красивые текстуры;
- стильные кнопки и панели;
- визуально дорогой, собранный, не перегруженный UI.

Интерфейс должен ощущаться как премиальный личный блокнот для iPad, а не как обычный веб-интерфейс.

==================================================
ГЛАВНАЯ СТРАНИЦА
==================================================

Переделай главный экран.

Структура:
- сверху по центру крупный заголовок: "Мои блокноты";
- ниже кнопки:
  - "Создать"
  - "Настроить фон приложения"
- ниже отображаются все созданные блокноты.

Но сами блокноты на главной странице должны выглядеть не как обычные карточки сайта, а как реальные блокноты.

Для каждого блокнота на главной странице нужно показывать:
- прямоугольную форму блокнота;
- цвет;
- текстуру/структуру обложки;
- внешний вид обложки;
- переплет, если он выбран;
- кольца, пружину или другой визуальный признак переплета, если это выбрано.

На самом блокноте по центру должно быть:
- только название блокнота, если пользователь его задал;
- если названия нет, то на обложке ничего не писать.

На главной странице НЕ нужно отображать:
- тип блокнота;
- тип бумаги;
- выбранную ручку;
- служебные подписи;
- нижние названия под карточками;
- лишние метаданные.

Главная страница должна восприниматься как набор реальных блокнотов, а не как список карточек с технической информацией.

НАСТРОЙКА ФОНА ПРИЛОЖЕНИЯ
==================================================

Настройка фона приложения не должна быть просто выбором цветов.

Сделай настройку фона через визуальные карточки-превью с реальными изображениями сцен.

Пользователь должен иметь возможность:
- выбрать фон из готовых пресетов;
- загрузить собственное изображение для фона приложения;
- сохранить пользовательский фон локально;
- восстановить его после перезагрузки.

Исправь и реализуй такие фоновые пресеты как полноценные сцены, а не как абстрактные цветовые оттенки:

1. "Глубокий космос"
Должно быть видно:
- звезды;
- туманности;
- глубина космоса.

2. "Метеоритный поток"
Должно быть видно:
- метеориты;
- космос;
- ощущение потока метеоров.

3. "Ночная трасса"
Должно быть видно:
- ночная дорога/трасса;
- звездная ночь;
- ночное небо.

4. "Северный двигатель"
Должно быть видно:
- северное сияние;
- элементы двигателя/механики на фоне.

5. "Пыль Сатурна"
Должно быть видно:
- планета Сатурн;
- кольца Сатурна;
- космическая пыль;
- космический мусор.

6. "Карбоновый дрифт"
Должен быть фон, реально соответствующий названию:
- карбон;
- динамичный технический стиль;
- мото/авто настроение;
- не просто серо-черный градиент.

Добавь отдельные настройки:
- сила затемнения фона;
- сила размытия фона.

Пользователь должен иметь возможность:
- уменьшать или усиливать затемнение;
- уменьшать или усиливать blur/размытие;
- настраивать видимость фона так, чтобы он не мешал читаемости интерфейса.

Если нужно, используй overlay поверх фонового изображения.

- никаких лишних надписей на странице.

То есть пользователь должен сразу попадать в рабочее пространство блокнота.

СТРАНИЦА ДОЛЖНА ВЫГЛЯДЕТЬ КАК ЛИСТ БЛОКНОТА
==================================================

Открытая страница должна выглядеть как лист реального блокнота, а не как веб-страница.

Требования:

- страница не должна прокручиваться вниз как сайт;
- вертикальный бесконечный скролл убрать;
- новая страница появляется через перелистывание;
- новые страницы наследуют формат первой страницы;
- пользователь должен ощущать работу с листами, а не с веб-контентом.

ЛЕВАЯ БОКОВАЯ ПАНЕЛЬ
==================================================

Внутри открытого блокнота сделай постоянную левую вертикальную панель.

Требования:
- располагается слева;
- идет сверху вниз;
- овальной или сильно скругленной формы;
- занимает примерно одну шестую ширины экрана;
- не закрывает рабочую область листа;
- состоит только из значков;
- без подписей под значками;
- компактная, минималистичная;
- удобная для iPad;
- по духу интерфейса ближе к Procreate: ничего лишнего, быстрый доступ к функциям, рабочая область в приоритете.

Когда пользователь нажимает на значок:
- открывается список функций, относящихся к значку;
- повторное нажатие на тот же значок сворачивает список;
- интерфейс не должен быть постоянно раскрыт и захламлять экран;
- по возможности активным должен быть только один раскрытый блок за раз.

Если следующей страницы еще нет:
- она создается автоматически;
- наследует формат текущего блокнота:
  - тип бумаги;
  - цвет бумаги;
  - общий стиль страницы.

Важно:
- не использовать тяжелую анимацию, если она мешает производительности;
- приоритет — стабильность, плавность и ощущение листания.
